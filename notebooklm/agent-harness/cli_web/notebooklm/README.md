# cli-web-notebooklm

> Generated by [CLI-Anything-Web](../../../../cli-anything-web-plugin/) from [notebooklm.google.com](https://notebooklm.google.com)

Agent-native CLI for Google NotebookLM — create notebooks, add sources, ask questions, and generate artifacts from the command line.

## Installation

```bash
pip install -e .
```

## Auth

Login via browser (playwright-cli required):

```bash
cli-web-notebooklm auth login
```

Manual fallback (import cookies from JSON):

```bash
cli-web-notebooklm auth login --cookies-json cookies.json
```

Check status:

```bash
cli-web-notebooklm auth status
```

## Usage

### Interactive REPL

```bash
cli-web-notebooklm
```

### One-shot commands

**Notebook Context**

```bash
cli-web-notebooklm use <notebook-id>       # Set active notebook (supports partial ID)
cli-web-notebooklm status                   # Show active notebook and auth state
```

Once a notebook is set with `use`, the `--notebook` flag is optional on all commands.

**Notebooks**

```bash
cli-web-notebooklm notebooks list
cli-web-notebooklm notebooks list --json
cli-web-notebooklm notebooks create --title "My Research"
cli-web-notebooklm notebooks get <notebook-id>          # Supports partial ID
cli-web-notebooklm notebooks rename <notebook-id> --title "New Title"
cli-web-notebooklm notebooks delete <notebook-id> --confirm
```

**Sources**

```bash
cli-web-notebooklm sources list --notebook <notebook-id>
cli-web-notebooklm sources add-url --notebook <notebook-id> --url https://example.com/article
cli-web-notebooklm sources get <source-id> --notebook <notebook-id>
cli-web-notebooklm sources delete <source-id> --notebook <notebook-id> --confirm
```

**Chat**

```bash
cli-web-notebooklm chat ask --notebook <notebook-id> --query "What are the main topics?"
cli-web-notebooklm chat ask --query "Summarize key findings" --json   # uses active notebook
```

**Artifacts**

```bash
# Generate any artifact type
cli-web-notebooklm artifacts generate --type briefing --wait --output briefing.md
cli-web-notebooklm artifacts generate --type audio --wait --json
cli-web-notebooklm artifacts generate --type slide-deck --json

# List all artifacts with status
cli-web-notebooklm artifacts list --json

# Download a completed artifact
cli-web-notebooklm artifacts download <artifact-id> -o slides.pdf

# Other
cli-web-notebooklm artifacts generate-notes --retry 3
```

Artifact types: `audio`, `video`, `mindmap`, `study-guide`, `briefing`, `faq`, `quiz`, `infographic`, `slide-deck`, `data-table`

Options: `--wait` (poll until ready), `--retry N` (retry on rate limit), `--output FILE` (save content/download)

**User**

```bash
cli-web-notebooklm whoami
cli-web-notebooklm whoami --json
```

## JSON Output

Every command supports `--json` for machine-readable output. Errors also return structured JSON:

```bash
cli-web-notebooklm notebooks list --json | jq '.[].title'
cli-web-notebooklm chat ask --query "summarize" --json
```

```json
{"error": true, "code": "NOT_FOUND", "message": "Notebook abc123 not found"}
```

Error codes: `NOT_FOUND`, `NETWORK_ERROR`, `AUTH_REQUIRED`, `RATE_LIMITED`, `PARSING_ERROR`, `SERVER_ERROR`

## CI / CD

Set the `CLI_WEB_NOTEBOOKLM_AUTH_JSON` environment variable to a JSON string containing auth cookies. This bypasses browser-based login for headless environments.

```bash
export CLI_WEB_NOTEBOOKLM_AUTH_JSON='{"cookies": [...]}'
cli-web-notebooklm auth status --json
```
